5. How Konstance Self-Edits Its Own Code

When Xavier sends a plain-language message in Telegram:

Xavier:    'add rate limiting so users can only message once every 3 seconds'

Konstance: 1. Sends normal LLM reply first
           2. intent_router detects: is_code_request=true, risk=high
           3. Asks LLM to generate the full updated file
           4. Runs compile_check_string() on the generated code
           5. If compile fails ? reports error, nothing touched
           6. If HIGH RISK ? backup + store in pending_edits.json
              ? 'Staged as draft abc12345. /approve abc12345 to apply'
           7. Xavier sends: /approve abc12345
           8. apply_patch() runs: backup + write + post-compile
           9. self_model.refresh() � Konstance updates its self-knowledge
          10. 'Applied. Backup: bot.py.bak.1741234567. Test it now.'

6. How Konstance Knows Itself

On every single message, before the LLM sees the user's text, it receives:

SELF-MODEL BLOCK (injected into every prompt)
IDENTITY:         Name, owner, mission statement from mission.json
PERSONALITY:      Tone and communication style from mission.json
FINANCIAL CONTEXT: Why it exists and what outcomes matter
GOALS:            Numbered list of objectives from mission.json
CONSTRAINTS:      Hard rules it must never violate
MY CODEBASE:      Every .py file with its description (live scan)
MY COMMANDS:      Every /command currently registered
SELF-EDIT CAPABILITY:  What it can and can't modify without approval

The mission.json file is the control panel. Xavier edits it in Notepad � no code changes needed � and Konstance immediately reflects the updated goals on the next message.
